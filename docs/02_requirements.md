# Smart Notes — требования (MVP)

## MUST-требования

### MUST-1. Создание заметки
Приложение должно позволять пользователю создать новую заметку, введя текст и нажав кнопку добавления.

**Критерий приёмки:**  
Если пользователь вводит непустой текст и нажимает кнопку **Добавить**, новая заметка появляется в списке без перезагрузки страницы.

### MUST-2. Валидация ввода
Приложение не должно создавать пустую заметку или заметку, состоящую только из пробелов.

**Критерий приёмки:**  
Если пользователь пытается добавить пустую заметку, приложение не добавляет запись и показывает понятное сообщение об ошибке.

### MUST-3. Автоматическая категоризация
При создании заметки приложение должно автоматически назначать ей одну из категорий: **Учёба**, **Покупки**, **Личное**, **Другое**.

**Критерий приёмки:**  
Если текст заметки содержит ключевые слова категории, после сохранения заметка отображается с автоматически определённой категорией.

### MUST-4. Просмотр списка заметок
Приложение должно отображать список всех сохранённых заметок с текстом, категорией и датой создания.

**Критерий приёмки:**  
После запуска приложения пользователь видит ранее сохранённые заметки, если они уже есть в localStorage.

### MUST-5. Фильтрация по категории
Приложение должно позволять отфильтровать заметки по выбранной категории.

**Критерий приёмки:**  
При выборе категории в фильтре на экране остаются только заметки этой категории; при выборе "Все" снова показываются все заметки.

### MUST-6. Удаление заметки
Приложение должно позволять удалить заметку из списка.

**Критерий приёмки:**  
После нажатия на кнопку удаления соответствующая заметка исчезает из интерфейса и не появляется после перезапуска страницы.

### MUST-7. Локальное сохранение данных
Приложение должно сохранять заметки и настройки пользователя в localStorage.

**Критерий приёмки:**  
После закрытия и повторного открытия приложения заметки и выбранные настройки остаются сохранёнными.

### MUST-8. Адаптивный интерфейс
Интерфейс должен быть удобен для использования на экранах мобильных устройств и настольных компьютеров.

**Критерий приёмки:**  
На ширине экрана 360 px и на ширине 1280 px основные элементы интерфейса остаются доступными, читаемыми и не выходят за границы экрана.

## NICE-требования

### NICE-1. Ручное редактирование категории
Пользователь может вручную изменить автоматически назначенную категорию.

### NICE-2. Поиск по тексту
Пользователь может искать заметки по части текста.

### NICE-3. Переключение темы интерфейса
Пользователь может выбирать светлую и тёмную тему.

### NICE-4. Установка как PWA
Приложение можно установить на устройство из браузера.

### NICE-5. Закрепление важных заметок
Пользователь может помечать отдельные заметки как важные и отображать их выше остальных.
