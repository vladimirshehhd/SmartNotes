# Smart Notes — конфигурация, хранение данных, ограничения и политика ошибок

## 2. Где храним данные
Используется `localStorage`.

Логические сущности:
- `smart_notes_notes` — JSON-массив заметок;
- `smart_notes_settings` — JSON-объект настроек.

Логически они эквивалентны:
- `notes.json`
- `settings.json`

## 3. Политика кеша
### Runtime UI cache
Во ViewModel хранится текущее состояние списка в памяти.

### Offline shell cache
Service worker кеширует:
- `index.html`
- `style.css`
- `app.js`
- `manifest.json`

Ключ кеша: `smart-notes-shell-v1`.

## 4. Ограничения/лимиты
1. localStorage ограничен объёмом браузера.
2. Целевой объём MVP — до **1000 заметок**.
3. Максимальная длина одной заметки — **280 символов**.
4. В приватном режиме браузера localStorage может быть ограничен.

## 5. Политика ошибок

### Что показываем пользователю
- `Введите текст заметки.`
- `Слишком длинная заметка. Максимум — 280 символов.`
- `Не удалось загрузить сохранённые данные.`
- `Не удалось сохранить изменения. Проверьте настройки браузера.`
- `Произошла ошибка. Попробуйте ещё раз.`
