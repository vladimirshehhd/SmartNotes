# Smart Notes — Оценка нагрузки / лимитов

## 1. Сценарии нагрузки

### S1. Добавление новой заметки
- сетевых запросов: **0**
- операций localStorage: до **1 чтение + 1 запись**

### S2. Переключение фильтра 10 раз подряд
- сетевых запросов: **0**
- localStorage: **0**
- фильтрация выполняется по данным в памяти

### S3. Удаление 10 заметок подряд
- сетевых запросов: **0**
- localStorage: до **10 записей**

### S4. Повторное открытие приложения
- загрузка статических файлов GitHub Pages
- localStorage: **2 чтения** на старт

## 2. Оценка частоты операций
Худший реалистичный сценарий за 10 минут:
- 30 добавлений заметок;
- 10 удалений;
- 20 переключений фильтра.

Итог:
- чтения localStorage: примерно **30**
- записи localStorage: примерно **40**
- фильтрации в памяти: **20**

## 3. Меры защиты
1. Не читать localStorage при каждом рендере.
2. Хранить текущее состояние списка в памяти ViewModel.
3. Ограничить длину заметки до 280 символов.
4. Добавить debounce для UI-обновлений.
5. Блокировать повторную обработку кнопки **Добавить** на короткое время.
6. Минимизировать полные перерисовки DOM.

## 4. Оценка производительности UI
1. Добавление заметки при количестве до 500 записей — **<= 200 мс**.
2. Переключение фильтра при количестве до 500 записей — **<= 150 мс**.
3. Старт приложения с localStorage — **<= 500 мс** без учёта сетевой загрузки страницы.

## 5. Вывод
Проект не использует внешнее API, не спамит сеть и имеет низкие риски по производительности.
