# Smart Notes — нефункциональные требования

## 1. Производительность и отзывчивость UI
1. Добавление заметки должно отображаться в интерфейсе не более чем за **200 мс** после нажатия кнопки на типичном современном устройстве.
2. Фильтрация списка заметок до **500 записей** должна выполняться не более чем за **150 мс**.
3. Первичная загрузка приложения должна быть визуально завершена не более чем за **2 секунды** при обычной загрузке страницы из GitHub Pages и стабильном подключении.

## 2. Надёжность
1. Приложение не должно завершаться ошибкой при некорректном вводе пользователя.
2. При ошибке чтения или записи localStorage приложение должно показать понятное сообщение и продолжить работу в деградированном режиме, если это возможно.
3. Все пользовательские ошибки должны сопровождаться понятным текстом без технического жаргона.

## 3. Хранение данных
1. Заметки и настройки должны храниться локально в браузере пользователя через localStorage.
2. После обновления страницы данные должны сохраняться.
3. Формат хранения должен быть JSON-совместимым и версионируемым.

## 4. Ограничения платформы
1. Поддерживаемые платформы: современные браузеры на Windows, Linux, macOS, Android.
2. Минимально поддерживаемые браузеры: Chrome 110+, Edge 110+, Firefox 110+.
3. Приложение не требует прав администратора, установки драйверов и серверной части.
4. Для PWA-функций браузер должен поддерживать service worker и manifest.

## 5. Безопасность и приватность
1. Приложение не должно отправлять заметки пользователя на внешний сервер.
2. В репозитории не должны храниться секреты, токены, API-ключи и персональные данные.
3. Приложение должно экранировать пользовательский ввод при отображении, чтобы не допускать XSS через содержимое заметок.
4. Все данные считаются локальными и доступны только в браузере пользователя на его устройстве.
