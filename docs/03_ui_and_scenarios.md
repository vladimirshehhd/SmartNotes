# Smart Notes — UX-анализ, интерфейс, сценарии и ошибки

## 1. Список экранов

### 1.1. Главное окно
Главный экран предназначен для повседневной работы с заметками:
- ввод новой заметки;
- просмотр списка заметок;
- фильтрация по категориям;
- удаление заметок;
- переход к настройкам.

### 1.2. Экран настроек
Экран настроек нужен для управления базовыми параметрами приложения:
- тема интерфейса;
- категория по умолчанию для неопределённых заметок;
- очистка всех данных;
- просмотр краткой информации о приложении.

## 2. Элементы на каждом экране

### 2.1. Главное окно
1. Заголовок приложения **Smart Notes**.
2. Текстовое поле для ввода заметки.
3. Кнопка **Добавить**.
4. Блок с краткой подсказкой по автокатегоризации.
5. Фильтры: **Все**, **Учёба**, **Покупки**, **Личное**, **Другое**.
6. Список карточек заметок.
7. На карточке заметки:
   - текст заметки;
   - категория;
   - дата/время создания;
   - кнопка удаления.
8. Кнопка перехода в настройки.

### 2.2. Экран настроек
1. Заголовок **Настройки**.
2. Переключатель темы: светлая / тёмная.
3. Выбор категории по умолчанию для заметок без совпадений.
4. Кнопка **Очистить все заметки**.
5. Кнопка **Назад**.
6. Блок **О приложении**.

## 3. Состояния UI

### Empty
Список заметок пуст.  
Показывается сообщение: **"Заметок пока нет. Добавьте первую запись."**

### Loading
Используется при старте приложения и чтении localStorage.  
Текст: **"Загрузка..."**

### Success
Данные успешно загружены или заметка успешно создана/удалена.

### Error
Произошла ошибка валидации или ошибка доступа к хранилищу.  
Показывается сообщение пользователю рядом с формой или в виде уведомления.

## 4. Пользовательские сценарии

### Сценарий 1. Добавление новой заметки
1. Пользователь открывает приложение.
2. Вводит текст: **"подготовить доклад по ИИ"**.
3. Нажимает кнопку **Добавить**.
4. Приложение определяет категорию **Учёба**.
5. Новая заметка появляется в списке.

### Сценарий 2. Фильтрация по категории
1. Пользователь открывает список заметок.
2. Выбирает фильтр **Покупки**.
3. Приложение показывает только заметки категории **Покупки**.

### Сценарий 3. Удаление заметки
1. Пользователь находит неактуальную заметку.
2. Нажимает кнопку удаления.
3. Приложение удаляет запись из интерфейса и localStorage.
